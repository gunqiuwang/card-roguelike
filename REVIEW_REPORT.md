## 审查维度分析

### 1. 是否真的提升体验？✅ YES
- **斩妖派新增"斩击狂热"机制**：连续出斩妖攻击牌，伤害+3/+6/+9...
- **视觉反馈强化**：combo ≥ 2 时触发斩妖特效文字
- **状态提示**：手牌区显示当前斩击层数和累计加成伤害
- **体验升级**：玩家有明确的"叠斩"动机和视觉爽感

### 2. 是否破坏已有玩法？✅ NO
- **向后兼容**：只给斩妖攻击牌添加 `comboBonus` 标记
- **不改变原有卡牌数值**：所有 value、cost、rarity 保持不变
- **回合重置机制**：`START_TURN` 时 `zhanyaoCombo` 归零，防止无限滚雪球
- **非强制机制**：玩家可自由选择是否叠斩，不影响游戏通关

### 3. 是否有假优化（只改UI颜色）？✅ NO
- **核心机制真实改动**：`gameStore.ts` 中伤害计算逻辑真实修改
- **数值真实增长**：`damage += comboCount * 3` 是真实伤害加成
- **动画反馈真实**：不是简单颜色变化，而是 `zhanyaoRage` 特效动画

### 4. 是否符合移动端体验？✅ YES
- **触控友好**：叠斩是自然行为，不需要额外点击
- **性能安全**：动画使用 CSS `drop-shadow` 和 `scale`，无 heavy GPU 负担
- **屏幕适配**：特效字体使用 `rem` 单位，自动适配不同屏幕
- **Bundle 增量小**：新增代码约 3KB，Build 从 190KB → 192KB（+2KB）

### 5. 是否引入风险？⚠️ MINOR
- **潜在问题**：`zhanyaoCombo` 字段是新增字段，首次 load 会是 `undefined`
- **已做防护**：`gameStore.ts` 中使用 `|| 0` 处理 undefined 情况
- **风险等级**：极低（不影响游戏正常运行）
