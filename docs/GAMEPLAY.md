> 版本：v0.1 · 2026.05
> 本文档是所有玩法实现的唯一规格。代码行为与此文档不一致时，改代码或改文档，不得默许。

## 一、三层爽感结构

```
┌──────────────────────────────────────────┐

├──────────────────────────────────────────┤
```

## 二、核心循环（Core Loop）

一局完整流程：

```
标题页 → 选职业（方士/阴阳师*）→ 进入第一境
  │
  └─→ 节点地图（类 StS）
       │
       ├─→ 战斗节点
       │    │
       │    ├─→ 打出手牌 → 敌人 HP 降低
       │    ├─→ 敌人血量 ≤ 30% → 弹出【斩】/【封】选择
       │    │    ├─ 斩 → 获得灵气 + 卡牌奖励（从池抽 3 选 1）
       │    │    └─ 封 → 拼符小游戏 → 成功则该妖成为新卡入组
       │    └─→ 战斗结算
       │
       ├─→ 祭坛节点（商店）
       │    ├─ 消耗灵气：驯妖（降妖性）/ 升级卡 / 删卡
       │    └─ 卖秘卷（被动 buff）
       │
       ├─→ 奇遇节点（事件）
       │    └─ 文字抉择，影响妖性/资源/命运
       │
       └─→ Boss 节点（每境末尾）
            └─ Boss 战 → 胜 → 进入下一境
  │
  └─→ 第五境混沌 → 终局选择（镇封 / 噬主）→ 结算 → 解锁 meta
```

*v0.1 方士为唯一职业

## 三、战斗系统（基础）

### 3.1 回合制

- 每回合玩家先行，敌人后行
- 玩家每回合：抽 5 张、能量 3 点
- 手牌上限 10，牌库用完洗弃牌堆

### 3.2 能量 / 灵气 / HP

| 资源 | 作用 | 恢复 |
|---|---|---|
| **能量（气）** | 出牌消耗，每张卡 0~3 费 | 每回合开始满 |
| **灵气** | 战后货币，祭坛用 | 击败敌人获得 |
| **HP（气血）** | 死亡即 Game Over | 战后部分回复 / 祭坛 |
| **妖性（v0.4）** | 妖卡的隐藏条 | 使用 +，休眠 - |

### 3.3 卡牌类型

| 类型 | 说明 | 例 |
|---|---|---|
| **符咒** | 主动法术，有费消耗 | 烈焰符、镇魂符 |
| **法器** | 被动/触发类 | 八卦镜、桃木剑 |
| **妖卡** | 封印获得，带妖性 | 九尾幻术、相柳毒噬 |

### 3.4 敌人意图（Intent）

每回合敌人行动前显示意图图标：
- ⚔️ 攻击（显示伤害数）
- 🛡️ 防御
- 🌀 蓄力（N 回合后大招）
- ✨ 施法（状态效果）
- ❓ 未知（Boss 特殊）

## 四、第一层钩子：**封妖系统**（v0.2 实现）

### 4.1 触发条件

- 敌人 HP ≤ 30% 时
- 敌人头顶出现**镇妖印**图标（闪烁提示）
- 玩家下一次会对该敌造成击杀伤害时，弹出选择：
  - **【斩】**：正常击杀，获得灵气 + 卡牌奖励（3 选 1）
  - **【封】**：进入拼符小游戏（v0.3 之前为一键成功）

### 4.2 封印成功

- 该妖**变成一张卡**加入牌组
- 妖卡的属性从妖怪的**本体能力**派生（详见 `src/data/yao/*.ts` 转换规则）
- 例：九尾狐（敌人能力：幻术，每 2 回合让玩家丢弃 1 张牌）
  - → 妖卡《九尾幻术》：1 费，让敌人下回合丢弃 1 个意图（使其空过）
- 牌组上限：**20 张**。超限时从牌组选 1 张卡弃置。

### 4.3 封印失败（v0.3 拼符引入后才有）

- 拼符失败 → 妖逃走，不获得妖卡，但仍获得少量灵气
- 失败次数不影响该局其他战斗

### 4.4 封印数值设计

| 妖等级 | 作为敌人的强度 | 封后的妖卡强度 | 妖性初始值 |
|---|---|---|---|
| C 普通 | 弱 | 等同于起手卡 | 10 |
| B 精英 | 中 | 稀有卡水平 | 30 |
| S Boss | 强 | 传说卡水平 | 50 |

## 五、第二层钩子：**拼符系统**（v0.3 实现，v0.1 仅预留接口）

### 5.1 基础概念

每张**封印符阵**由 3-5 个**笔画元素**组成，玩家需要在限定时间内**按正确顺序点击/拖拽**笔画。

**笔画元素**（只有这 6 种）：

| 元素 | 符号 | 含义 | 获取方式 |
|---|---|---|---|
| 点（丶）| `・` | 起始、封存 | 每局默认 |
| 横（一）| `—` | 束缚、压制 | 每局默认 |
| 竖（丨）| `｜` | 贯穿、封锁 | 每局默认 |
| 折（乚）| `乚` | 转化、阴阳 | 精英妖后解锁 |
| 钩（丿）| `丿` | 勾引、诱导 | 秘卷解锁 |
| 回（囗）| `囗` | 镇压、终结 | Boss 后解锁 |

### 5.2 封印符阵

每只妖有**一个专属符阵**（从易到难）：

```
例：青狐小妖 → 符阵「封狐」
笔顺：丶 → 一 → 丨
难度：★☆☆

例：九尾狐 → 符阵「九镜封」
笔顺：丶 → 丶 → 丶 → 一 → 乚 → 囗
难度：★★★☆
```

### 5.3 拼符结果

| 完成度 | 条件 | 结果 |
|---|---|---|
| **完美** | 顺序全对 + 时间 < 80% | 获得妖卡 + 该卡**升阶版** |
| **成功** | 顺序全对 | 获得妖卡 |
| **失败** | 顺序错 / 超时 | 妖逃走 + 少量灵气 |

### 5.4 难度曲线

- **第一境**：只用前 3 种笔画（点横竖）
- **第二境**：解锁"折"
- **第三境**：解锁"钩"
- **第四境**：解锁"回"
- **第五境**：混合符阵（一次 6-8 笔）

## 六、第三层钩子：**妖性系统**（v0.4 实现，v0.1 预留）

### 6.1 妖性值

每张妖卡有 **0~100** 的妖性值：

```
[0-30]    : 温顺  → 正常使用
[31-60]   : 躁动  → 打出时 10% 概率自残 1
[61-90]   : 狂乱  → 打出时必自残，且 30% 概率逼迫弃牌
[91-100]  : 噬主  → 无法使用，下回合开始自残 5 直至"镇压"
```

### 6.2 妖性变化

- 打出该妖卡 → 妖性 +5
- 该妖卡在弃牌堆过一轮 → 妖性 -1
- 祭坛"驯妖" → 花灵气降妖性
- 某些事件/Boss 技能 → 强制增妖性

### 6.3 反噬事件（让玩家记忆深刻的一击）

当牌组总妖性 > 阈值时，**夜间事件**触发：
- "你在梦中听见万妖低语..."
- 随机：失去 HP / 永久增一张"妖影"废卡 / 丢失一件法器
- 目的：**让玩家感到"代价"，而不只是"收集"**

## 七、第四层：**阴阳双道职业**（v0.5，暂不实现）

### 7.1 解锁条件

- 任意职业通关 1 次
- 进入"阴阳录"meta 场景

### 7.2 玩法差异（方士 vs 阴阳师）

| 维度 | 方士 | 阴阳师 |
|---|---|---|
| 手牌 | 5 张单列 | 上（阳）3 张 + 下（阴）3 张 |
| 能量 | 3 点 | 阳 2 + 阴 2 |
| 核心机制 | 符咒链 | 阴阳平衡 |
| 大招 | 无 | "太极归一" |

详见未来 `docs/YINYANG.md`。

## 八、Meta / 长线进度

### 8.1 图鉴（v0.4）

- 每只封过的妖 → 图鉴解锁一页
- 页面信息：原典条目、第一次封印时间、使用次数、反噬次数、flavor text
- 图鉴进度 → 解锁新难度、新事件、新隐藏妖

### 8.2 世相（周目 Modifier）

通关后开启"二周目"，可选一个"世相"：
- **阴年**：所有妖性增长 ×2
- **枯纪**：灵气收益 ×0.5，卡牌奖励 ×2
- **雾劫**：敌人意图隐藏
- 等等

## 九、数值平衡护栏

所有数值变更必须通过以下验收：

| 层级 | 检查 | 目标 |
|---|---|---|
| L1 单卡 | 期望伤害 / 回合 | [3.5, 5.5] |
| L2 构筑 | 10 套代表卡组跑 500 局胜率 | [40%, 70%] |
| L3 封妖 | 每局封妖数 | 3-5 只 |
| L4 节奏 | 普通战回合数 | 3-5 |
| L4 节奏 | Boss 战回合数 | 6-10 |

仿真器（v0.2 开始实现）：`src/sim/` 模块，每次 PR 自动跑 1000 局基准测试。

## 十、v0.1 不做什么（明确作用域）

**v0.1（本 PR）只做**：
- 项目骨架（目录、类型、配置）
- 视觉样机页（色板/字体/卡面/按钮/血条）
- 标题页（带立绘槽位）
- 一份示例卡牌数据（3 张示例符咒 + 1 只示例妖）

**v0.1 不做**：
- ❌ 战斗逻辑（留到 v0.2）
- ❌ 地图节点（留到 v0.2）
- ❌ 拼符（v0.3）
- ❌ 妖性（v0.4）
- ❌ 阴阳（v0.5）

**v0.1 目标**：让用户**看到风格、感受质感、验证协作**。不是做游戏。
