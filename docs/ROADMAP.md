## v0.3 · 拼符系统 ✅ 已上线

**目标**：核心钩子 #2 —— 封印有操作感。

- [x] 封印小游戏 UI（笔画点击）
- [x] 6 种笔画元素 + 10 种符阵
- [x] 完美/成功/失败的差异化奖励
- [x] 随章节解锁更多笔画

**v0.3 已实现内容**：
- 每笔画倒计时（普通 3s，Boss/精英 1.5s）
- 10 种符阵 SVG 背景，笔画渐进绘制动画
- 笔画正确/错误视觉反馈（闪绿/抖红）+ 提示笔画金色脉冲
- 完美封印（限时全对无失误）→ 升阶妖卡 +3 伤害
- 失败：扣 8HP，妖恢复 50% HP，可重新触发封印

## v0.4 · 妖性系统 + 图鉴（开发中）

**目标**：长线收集欲 + 使用代价。

- [ ] 妖卡的妖性值（0-100）
- [ ] 反噬事件（噩梦场景）
- [ ] 祭坛"驯妖"
- [ ] 图鉴页面（封过的妖档案）
- [ ] 解锁式 meta 进度

**设计要点**：
- 妖卡打出时妖性 +5，闲置时每战斗 -1
- 妖性 ≥ 60 躁动（10% 自残 3），≥ 90 狂乱（必自残 5 + 25% 弃 1）
- 祭坛消耗灵气驯妖（-20 妖性 / 次）
- 图鉴按章节解锁，记录已封印妖的档案

## 发布流程

1. PR 合并 `main` → Vercel 自动部署预览
2. 手动验证 → 合并 `gh-pages` 发布静态页
3. GitHub Release 打版本 tag
