## 一 · 核心循环 · 一场战斗的精确流程

### 1.1 状态机

```
            ┌─────────────────┐
            │  BATTLE_START   │  初始化：洗牌、抽手牌、设置敌意图
            └────────┬────────┘
                     ▼
            ┌─────────────────┐
     ┌─────▶│  PLAYER_TURN    │  玩家回合开始
     │      │  · 能量 = 3      │
     │      │  · 抽 5 张        │
     │      │  · 气·御 归零     │
     │      └────────┬────────┘
     │               │
     │               ▼
     │      ┌─────────────────┐ ◀───┐
     │      │ PLAYER_ACTION   │     │ 循环：直到
     │      │ · 出牌 / 结束回合 │     │ · 能量不足
     │      └────────┬────────┘     │ · 手牌用完
     │               │              │ · 玩家主动结束
     │               ├──────────────┘
     │               │ 敌 HP ≤ 30% 时？ Yes ▶ SEAL_CHOICE
     │               │
     │               ▼（玩家主动结束）
     │      ┌─────────────────┐
     │      │  ENEMY_TURN     │  敌执行意图 → 对玩家结算
     │      └────────┬────────┘
     │               │
     │               ▼
     │      玩家 HP > 0 ?
     │         │          │
     │        Yes        No
     │         │          │
     └─────────┘          ▼
                  ┌─────────────────┐
                  │   GAME_OVER     │
                  └─────────────────┘
                  
                  
【斩/封分支】
┌─────────────────┐
│  SEAL_CHOICE    │ 弹出 UI：[斩] [封] [取消本次击杀（防误触）]
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
   【斩】     【封】
    │         │
    ▼         ▼
 KILL      SEAL_SUCCESS
 · 给灵气    · 妖卡加入牌组（进弃牌堆，战后入组）
 · 3 选 1    · 不给卡牌奖励
 · 敌 HP=0   · 不给灵气（或给少量）
    │         │
    └────┬────┘
         ▼
 ┌─────────────────┐
 │  BATTLE_WIN     │
 └─────────────────┘
```

### 1.2 回合内详细动作

**玩家回合开始（PLAYER_TURN_START）**：
1. 气·御（block）归零 ← 关键，不继承
2. 能量重置为 `energyMax`（默认 3）
3. 抽 `handSize` 张（默认 5）；牌库不够时洗弃牌堆
4. 触发所有"回合开始"类 hook（暂无）

**玩家出牌（PLAY_CARD）**：
1. 检查能量 ≥ 卡费 → 否则卡片灰色不可点
2. 扣能量
3. 执行卡牌效果（见 §4 效果标准库）
4. 卡进**弃牌堆**（特殊："消耗"类进**消耗堆**，"蜕变"类进牌组）
5. 检查敌人 HP：若 ≤ 30% 且未触发过 SEAL_CHOICE → 暂停，弹斩/封
6. 若敌 HP = 0 且无 SEAL_CHOICE 机会 → 直接进 BATTLE_WIN

**玩家主动结束回合（END_TURN）**：
- 剩余能量不保留
- 手牌**保留**（不强制丢，v0.2 简化）
  - 注：StS 是丢手牌 → v0.2 保留是为了让节奏松一点，v0.3 验收后决定是否改

**敌人回合（ENEMY_TURN）**：
1. 按当前意图执行（攻击 → 扣玩家 HP；防御 → 敌 block；蓄力 → 回合数 -1）
2. 更新下一回合的意图（按敌 AI 脚本）
3. 移交回合给玩家

### 1.3 结束条件

| 条件 | 结果 |
|---|---|
| 敌 HP = 0 (KILL) | BATTLE_WIN，给灵气 + 卡牌奖励 |
| 敌被封 (SEAL) | BATTLE_WIN，给妖卡（弃牌堆进）+ 少量灵气，无卡牌奖励 |
| 玩家 HP = 0 | GAME_OVER |

## 二 · 资源系统

### 2.1 三种资源的精确定义

| 资源 | 范围 | 回合内 | 跨回合 | 跨战斗 | 归零条件 |
|---|---|---|---|---|---|
| **HP（气血）** | [0, maxHp] | 不自动回 | 保留 | 保留 | HP=0 → GAME_OVER |
| **能量（气）** | [0, energyMax] | 可用 | **不保留**，回合开始重置 | N/A | 回合结束自动丢弃 |
| **气·御（block）** | [0, ∞) | 抵伤 | **不保留**（玩家回合开始归零） | N/A | 抵挡完 |
| **灵气（currency）** | [0, ∞) | 战后货币 | 保留 | 保留 | 祭坛消耗 |

### 2.2 伤害结算公式

```
damage_to_hp = max(0, damage_incoming - block)
block_remaining = max(0, block - damage_incoming)

# 关键规则：先扣气·御，再扣 HP
```

**没有"先 HP 再 block" 反逻辑**。

### 2.3 关键常量（锁定）

```ts
// 对应 src/config/game.ts
handSizePerTurn = 5        // 每回合抽牌
handSizeMax = 10           // 手牌上限（暂无作用，预留）
energyPerTurn = 3          // 每回合能量
deckSizeStart = 10         // 起手牌组
deckSizeMax = 20           // 牌组上限（超了要弃卡）
maxHpStart = 70            // 方士起手最大 HP
sealThresholdHpPercent = 0.30   // 30% 血触发斩/封
```

### 2.4 起手牌组（固定 10 张，方士职业）

| 卡名 | 数量 | 费 | 效果 |
|---|---|---|---|
| 烈焰符 | 4 | 1 | 造成 6 伤害 |
| 镇魂符 | 4 | 1 | 获得 5 气·御 |
| 桃木斩 | 1 | 2 | 造成 8 伤害，若敌 HP ≤ 30% 额外 +4 |
| 抽符 | 1 | 1 | 抽 2 张 |

**为什么是这 4 张**：
- 保证每回合能打出至少 2 张普通牌（4+4+1+1 = 10 / 5 = 2 轮就能循环）
- 桃木斩 是"斩与封"的胁迫者（推血到 30%，让玩家面对选择）
- 抽符 是引擎卡（防止死手）

## 三 · 卡牌系统

### 3.1 卡片类型

| type | 说明 | 打出后去向 |
|---|---|---|
| **fu 符咒** | 主动法术（伤害/防御/抽牌/状态） | 弃牌堆 |
| **faqi 法器** | 同上，机械上等同符咒（语义分类） | 弃牌堆 |
| **yao 妖卡** | 封印获得，效果由原妖能力派生 | 弃牌堆 |
| 消耗类（exhaust=true 的字段） | 打出即入消耗堆，整局不回 | 消耗堆 |

### 3.2 卡牌数据结构（固化）

```ts
interface Card {
  id: string;                   // 'fu_fire_strike'
  name: string;                 // '烈焰符'
  type: 'fu' | 'faqi' | 'yao';
  rarity: 'starter' | 'common' | 'rare' | 'epic' | 'legend';
  school: 'zhanyao' | 'yuling' | 'fushu' | 'neutral';
  cost: number;                 // 0..3（X 费 = -1 预留）
  description: string;          // UI 显示
  flavor?: string;              // 山海经引文
  effects: Effect[];            // §4 效果数组，按序执行
  exhaust?: boolean;            // 打出即消耗
  ethereal?: boolean;           // 回合结束若未打出则消耗（v0.2 不用）
  artSrc?: string;
  silhouette?: SilhouetteKind;
  // v0.4+ 妖卡专属
  yaoxing?: number;             // 0..100 初始妖性
}
```

### 3.3 卡牌稀有度 × 出现率（战后 3 选 1）

| 稀有度 | 凡 common | 珍 rare | 灵 epic | 玄 legend |
|---|---|---|---|---|
| 普通战（C 妖） | 70% | 25% | 5% | 0% |
| 精英战（B 妖） | 30% | 50% | 18% | 2% |
| Boss 战（S 妖） | 0% | 30% | 55% | 15% |

## 四 · 效果标准库（Effect 类型）

**原则**：所有卡牌效果都由**原子 effect** 组合而成，不得写死在 Card.tsx 里。

### 4.1 原子 Effect 清单（v0.2 实现）

```ts
type Effect =
  | { kind: 'damage'; amount: number; target?: 'enemy' | 'all' }
  | { kind: 'block'; amount: number }
  | { kind: 'draw'; count: number }
  | { kind: 'gainEnergy'; amount: number }
  | { kind: 'applyStatus'; status: StatusKind; stack: number; target: 'enemy' | 'self' }
  | { kind: 'execute'; hpPercent: number; bonusDamage: number }  // 桃木斩的专属
  | { kind: 'sealIntent'; turns: number }  // 妖·九尾幻术：使敌下回合意图变为空过
  | { kind: 'discardEnemyBlock' }  // 破盾
```

### 4.2 状态（Status）清单

| StatusKind | 名称 | 效果 | 去除方式 |
|---|---|---|---|
| `poison` | 中毒 | 每回合开始 -1 HP / 层 | 每回合 -1 层 |
| `weak` | 虚弱 | 造成伤害 × 0.75 | N 回合 |
| `vulnerable` | 易伤 | 受到伤害 × 1.25 | N 回合 |
| `strength` | 力（Boss 专属） | +1 伤害 / 层（永久） | 无 |
| `intentSealed` | 意图封印 | 下回合意图强制空过 | 1 回合 |

### 4.3 关键公式

```
# 伤害计算（含虚弱/易伤修正）
final_damage = round(
  base_damage
  * (attacker.weak > 0 ? 0.75 : 1.0)
  * (target.vulnerable > 0 ? 1.25 : 1.0)
  + attacker.strength
)
```

## 五 · 敌人系统

### 5.1 敌人数据结构

```ts
interface Enemy {
  id: string;                 // 'qinghu'
  name: string;               // '青狐小妖'
  rank: 'C' | 'B' | 'S';
  chapter: 'qingqiu';
  hp: number;                 // 最大血
  intents: Intent[];          // 意图脚本（按回合循环）
  sealedCard: Omit<Card, 'id'>;  // 被封后变成的卡
  flavor: string;             // 被封前说的话
  artSrc?: string;
  silhouette: SilhouetteKind;
}

interface Intent {
  kind: 'attack' | 'defend' | 'charge' | 'cast';
  damage?: number;
  block?: number;
  status?: { kind: StatusKind; stack: number };
  label: string;  // UI 显示文案
}
```

### 5.2 意图 AI

- 敌有一个**固定意图序列**（数组 `intents`），按回合循环
- 每回合**公示下一回合要做什么**（玩家能看见）
- 示例：`[attack(5), defend(4), attack(5), charge, cast_big]` —— 5 回合一个循环

**v0.2 故意不做 AI**：固定脚本够了，玩家能学会预判；v0.3+ 再加"被斩击中会改变后续行为"之类。

### 5.3 Boss 的蓄力意图

Boss 有"**蓄力大招**"：
- 回合 N-2：意图显示 "🌀 蓄力中"
- 回合 N-1：意图显示 "🌀🌀 蓄力更强"
- 回合 N：释放大招（高伤害 / 全体状态）
- 玩家有**2 个回合**打断/准备（玩家可以打爆它的血进斩/封分支跳过大招）

## 六 · 封妖核心机制

### 6.1 触发时序（精确到帧）

```
玩家打出一张牌 → 结算伤害
   ↓
敌 HP 从 X 降到 Y
   ↓
if Y ≤ maxHp * 0.30 && !已触发过 SEAL_CHOICE：
  ↓
  设敌标记 markedForSeal = true
  显示"镇妖印"图标在敌头上（闪烁）
  ↓
  本回合后续所有可能"致死一击"的出牌暂停，弹 UI：
    【斩】 → 正常结算致死，给灵气 + 卡牌奖励
    【封】 → 封印动画，敌立刻 HP=0，妖卡进弃牌堆
    【暂不决定】 → 本次出牌取消，能量退还（防误触）
  
  一旦选择，markedForSeal 锁死，本场战斗不再弹
```

**关键**：**触发时机不是"HP ≤ 30%"那一刻**，而是"**玩家要打致死一击**"的那一刻。避免玩家先磨到 29% 再慢慢思考（这样会让节奏卡死）。

### 6.2 封印成功

```ts
function onSealSuccess(enemy: Enemy, gameState: GameState) {
  // 1. 敌 HP = 0，战斗进入 BATTLE_WIN
  enemy.hp = 0;
  
  // 2. 生成妖卡（clone，确保每场不污染）
  const yaoCard: Card = {
    ...enemy.sealedCard,
    id: `yao_${enemy.id}_${Date.now()}`,
    yaoxing: getInitialYaoxing(enemy.rank),  // C=10 B=30 S=50（v0.4 才用）
  };
  
  // 3. 卡进"本场弃牌堆"，战后整合进牌组
  gameState.discardPile.push(yaoCard);
  
  // 4. 给少量灵气（封妖也是有收获的，但不如斩）
  gameState.currency += Math.floor(enemy.maxHp * 0.15);  // C≈5 B≈15 S≈30
  
  // 5. 不给卡牌奖励（封妖本身就是卡牌奖励）
}
```

### 6.3 斩 vs 封 的经济差

| 选择 | 灵气奖励 | 卡牌奖励 | 加入牌组 | 策略含义 |
|---|---|---|---|---|
| 斩 | 100%（见 §7.2） | 3 选 1 | 选中的那张 | 资源流 / 自由构筑 |
| 封 | 15% | 无 | 妖卡 | 收集流 / 妖卡构筑 |

**设计意图**：**斩更灵活，封更专一**。玩家可以前期多斩攒资源，遇到喜欢/强的妖才封。不是"每个都封"的陷阱。

### 6.4 牌组上限处理（超 20 张）

**当封妖/卡牌奖励会让牌组 > 20 时**，强制弹出：
- 显示当前牌组
- 玩家**必须**选一张删除（可以删新加入的那张 = 相当于放弃）
- 无法跳过

## 七 · 战后奖励

### 7.1 斩的奖励流程

```
BATTLE_WIN (KILL 分支)
  → 灵气 +reward.currency
  → 卡牌奖励：展示 3 张（按 §3.3 稀有度概率抽）
  → 玩家选 1 张 or 跳过
  → 回血（见 §7.3）
  → 下个节点
```

### 7.2 灵气奖励表

| 节点类型 | 基础灵气 | 精英加成 | Boss 加成 |
|---|---|---|---|
| 普通战 C | 10~15（随机） | - | - |
| 精英战 B | 25~35 | ×1.5 | - |
| Boss 战 S | 50~60 | - | ×2.0 |

### 7.3 战后回血

- 普通战胜利：回血 **maxHp × 10%**（四舍五入）
- 精英战胜利：回血 **maxHp × 15%**
- Boss 战胜利：回血 **maxHp × 30%** + 最大 HP 永久 +5
- **封妖胜利**：回血 × 0.5（代价：你跟它签了契，耗神）

## 八 · 地图 & 节点

### 8.1 v0.2 的线性地图（第一章「青丘残岭」）

```
START
  │
  ▼
[战斗 1] 青狐小妖 × 1        ← 练手
  │
  ▼
[战斗 2] 青狐小妖 × 2        ← 多目标初体验
  │
  ▼
[奇遇] 荒坟                  ← 文字事件，3 选 1
  │
  ▼
[战斗 3] 青狐小妖 × 1 + 夜狐 × 1  ← 混编
  │
  ▼
[祭坛] 瞽人·祝               ← 商店
  │
  ▼
[精英] 九尾狐·绯（B 级）     ← 章中 Boss 预演
  │
  ▼
[战斗 4] 夜狐 × 2            ← 精英战后回气
  │
  ▼
[BOSS] 绯·九尾真身（S 级）   ← 章节 Boss
  │
  ▼
章末（v0.2 结束，v0.3 接第二章）
```

### 8.2 奇遇·荒坟（示例文字事件）

```
"你踏过一片坟岗，突然听见小孩的哭声从坟里传来。
 靠近，是一个穿白衣的女童，坐在坟上啃一只生鸡。
 她抬头看你，笑了。嘴角裂到耳根。"

[A] 斩她 → 女童化作青烟散去，获得 10 灵气
[B] 递给她一张符 → 她点点头，消失。弃牌堆获得一张"焚坟符"
[C] 转身离开 → 背后传来细碎笑声。手牌永久 -1 张（直到下次章节）
```

**事件原则**：
- 每个奇遇 3 个选项
- 必须有**一个明显代价选项、一个明显好处选项、一个神秘选项**
- 不设"正确答案"——玩家看选项感觉不出哪个最划算

### 10.1 存档格式

```ts
interface SaveData {
  version: 1;                   // SCHEMA_VERSION
  contentVersion: '0.2.0';      // 内容版本（卡/妖数据）
  runInProgress: RunState | null;

    runs: number;

  };

```

### 10.2 升级策略

- 加字段 → 默认值填充，向前兼容
- 删字段 → 跳过无视
- 重命名 / 类型变更 → **bump SCHEMA_VERSION**，写迁移函数
- 内容不兼容（如删了一只妖） → 清空 runInProgress，保留 meta

## 十二 · 风险与决策

**已知风险**：

1. **封印触发时机设计**：我选了"玩家要打致死那一刻"。如果体验发现节奏卡（比如玩家总是不敢打致死），改成"HP ≤ 30% 时就能**随时按键封印**"。
2. **牌组上限 20 太紧**：可能玩家根本不想封妖因为怕超上限。如果仿真器显示封妖数 < 2 → 放宽到 25 或去掉上限。
3. **战后回血 10%**：可能偏低。如果仿真器显示 Boss 战前玩家平均残血 < 40% → 提高到 15%。

**所有上述参数都在 `src/config/balance.ts`**，改数字不用改代码。
